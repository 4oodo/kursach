# Документация администратора - Система управления энергопотреблением

## Описание системы

Это приложение предназначено для администраторов и менеджеров энергосбережения. Система позволяет:

1. **Управлять инфраструктурой**: здания, помещения, категории
2. **Регистрировать энергопотребление**: вводить и отслеживать расход энергии
3. **Анализировать данные**: генерировать отчеты и статистику
4. **Выполнять пользовательские запросы**: через встроенную SQL консоль

---

## Установка и первый запуск

### Предварительные требования

- Windows 7 или новее
- .NET Framework 4.7.2 (или новее)
- SQL Server 2012 или новее
- Не менее 100 МБ свободного места на диске

### Пошаговая установка

#### Шаг 1: Подготовка SQL Server

1. Откройте **SQL Server Management Studio**
2. Подключитесь к вашему экземпляру SQL Server
3. Откройте файл `Database_Setup.sql` из папки приложения
4. Выполните скрипт (Ctrl+E или Query → Execute)
5. Дождитесь сообщения о успешном создании БД

#### Шаг 2: Настройка приложения

1. Отредактируйте файл `App.config`:
   ```xml
   <connectionStrings>
	   <add name="EnergyManagementDB" 
			connectionString="Data Source=YOUR_SERVER;Initial Catalog=EnergyManagement;..."
			providerName="System.Data.SqlClient" />
   </connectionStrings>
   ```

2. Замените `YOUR_SERVER` на имя вашего SQL Server сервера
   - Локальный сервер: `.` или `(local)`
   - Удаленный сервер: `SERVER_NAME` или `IP_ADDRESS`

#### Шаг 3: Запуск приложения

1. Запустите `kursach.exe` из папки `bin\Release\`
2. При успешном подключении откроется главное окно приложения

## Подробное описание функций

### 1. Вкладка "Управление зданиями"

**Назначение**: Ведение справочника зданий организации

**Поля**:
- `Название здания` - уникальное имя здания (обязательно)
- `Адрес` - физический адрес здания (обязательно)

**Операции**:
- **Добавить**: Введите данные → нажмите "Добавить"
- **Обновить**: Дважды кликните строку → измените данные → "Обновить"
- **Удалить**: Выберите строку → нажмите "Удалить" → подтвердите

**Примеры**:
```
Название: Главное административное здание
Адрес: ул. Центральная, 42
```

---

### 2. Вкладка "Управление категориями"

**Назначение**: Классификация типов помещений

**Встроенные категории**:
- Офис - рабочие помещения
- Коридор - проходные помещения
- Склад - хранилища
- Кафе - места общего питания
- Серверная - помещения IT инфраструктуры
- Лестничная клетка - вспомогательные помещения

**Как добавить новую категорию**:
1. Введите название категории
2. Нажмите "Добавить"

**Важно**: Категория не может быть удалена, если на нее ссылаются помещения!

---

### 3. Вкладка "Управление периодами"

**Назначение**: Определение временных интервалов для анализа потребления

**Стандартные периоды** (примеры):
```
Ночь:   00:00 - 06:00
Утро:   06:00 - 12:00
День:   12:00 - 18:00
Вечер:  18:00 - 00:00
```

**Другие варианты**:
- По смены (1-я смена, 2-я смена, 3-я смена)
- По дням недели (рабочие дни, выходные)

**Как использовать**:
Выбирайте период при регистрации энергопотребления

---

### 4. Вкладка "Управление помещениями"

**Назначение**: Каталог всех помещений в зданиях

**Процесс добавления помещения**:

1. Выберите здание из выпадающего списка
2. Введите номер этажа
3. Выберите категорию помещения
4. Нажмите "Добавить"

**Примеры заполнения**:
```
Здание: Главное здание
Этаж: 2
Категория: Офис
```

**Важно**: 
- Номер этажа должен быть положительным числом
- Используйте логическую нумерацию этажей (1, 2, 3... или -1 для подвала)

---

### 5. Вкладка "Управление энергопотреблением"

**Назначение**: Регистрация и отслеживание расхода энергии

**Процесс добавления записи**:

1. **Помещение**: Выберите помещение из списка
2. **Дата**: Выберите дату показания счетчика
3. **Период**: Выберите временной период
4. **Значение энергии (кВтч)**: Введите показание счетчика

**Пример**:
```
Помещение: Главное здание - Этаж 1
Дата: 2024-01-15
Период: День (12:00-18:00)
Значение: 245.50
```

**Советы**:
- Регистрируйте показания ежедневно в одно и то же время
- Используйте относительно точные измерения
- Проверяйте показания на ошибки перед сохранением

---

### 6. Вкладка "SQL консоль"

**Назначение**: Выполнение произвольных SQL запросов для опытных пользователей

**Как использовать**:

1. Введите SQL запрос в текстовое поле
2. Нажмите "Выполнить запрос"
3. Результаты отобразятся ниже

**Примеры полезных запросов**:

```sql
-- Получить все помещения в здании
SELECT * FROM dbo.Room 
WHERE BuildingID = 1

-- Общее потребление за день
SELECT SUM(EnergyValue) as TotalEnergy 
FROM dbo.EnergyConsumption 
WHERE [Date] = '2024-01-15'

-- Топ 10 "прожорливых" помещений
SELECT TOP 10 r.RoomID, b.BuildingName, SUM(ec.EnergyValue) as Total
FROM dbo.EnergyConsumption ec
JOIN dbo.Room r ON ec.RoomID = r.RoomID
JOIN dbo.Building b ON r.BuildingID = b.BuildingID
GROUP BY r.RoomID, b.BuildingName
ORDER BY Total DESC

-- Добавить новую запись
INSERT INTO dbo.EnergyConsumption (RoomID, [Date], TimePeriodID, EnergyValue)
VALUES (1, '2024-01-16', 2, 156.75)

-- Обновить запись
UPDATE dbo.Building 
SET Address = 'новый адрес' 
WHERE BuildingID = 1
```

**Предупреждение**: 
⚠️ SQL консоль позволяет выполнять любые операции с БД! Будьте осторожны с DELETE и DROP запросами!

### 7. Вкладка "Формирование отчетов"

#### Подвкладка "Отчет об энергопотреблении"

**Назначение**: Подробный анализ потребления энергии в заданном периоде

**Как использовать**:

1. Выберите **Дату начала** (например, 2024-01-01)
2. Выберите **Дату конца** (например, 2024-01-31)
3. Нажмите "Генерировать отчет"

**Информация в отчете**:
- Название здания
- Номер этажа
- Категория помещения
- Период времени
- Дата
- Значение потребления
- Кумулятивное потребление

**Применение**:
- Анализ тренда потребления энергии
- Выявление аномалий
- Отчеты для руководства

#### Подвкладка "Статистика по зданиям"

**Назначение**: Сводная статистика по всем зданиям

**Информация в отчете**:
- ID здания
- Название здания
- Количество помещений
- Общее энергопотребление
- Среднее энергопотребление
- Максимальное потребление
- Минимальное потребление

**Применение**:
- Сравнение эффективности зданий
- Выявление зданий-лидеров по потреблению
- Планирование мер энергосбережения

## Советы и лучшие практики

### 1. Регулярность ввода данных

- ✓ Вводите показания энергопотребления ежедневно
- ✓ Используйте одно и то же время для снятия показаний
- ✗ Не пропускайте дни (это создаст пробелы в данных)

### 2. Точность данных

- ✓ Проверяйте показания счетчиков перед вводом
- ✓ Используйте одни и те же единицы измерения (кВтч)
- ✗ Не вводите приблизительные значения
- ✗ Не копируйте вчерашние показания

### 3. Ведение справочников

- ✓ Регулярно обновляйте информацию о зданиях и помещениях
- ✓ Используйте последовательную нумерацию этажей
- ✗ Не удаляйте категории, которые уже используются

### 4. Резервное копирование

- Делайте резервные копии БД еженедельно
- Сохраняйте копии в безопасном месте
- Проверяйте целостность резервных копий

### 5. Анализ данных

- Проводите анализ потребления ежемесячно
- Сравнивайте данные с предыдущими периодами
- Выявляйте тренды и аномалии
- Докладывайте о значительных изменениях руководству

## Решение проблем

### Проблема: Не подключается к БД

**Решение**:
1. Проверьте, запущен ли SQL Server
2. Проверьте имя сервера в файле конфигурации
3. Проверьте параметры аутентификации
4. Убедитесь, что имеет место сетевое подключение

### Проблема: Таблица не создана

**Решение**:
1. Откройте SQL Server Management Studio
2. Выполните `Database_Setup.sql` заново
3. Проверьте наличие таблиц в базе

### Проблема: Ошибка при сохранении данных

**Решение**:
1. Проверьте, что все обязательные поля заполнены
2. Проверьте корректность введенных данных
3. Попробуйте операцию заново
4. Обратитесь к администратору системы
