# ✅ ОТЧЕТ О РЕАЛИЗАЦИИ ЛИЧНОГО КАБИНЕТА ПОЛЬЗОВАТЕЛЯ

## Задача
Создать отдельный кабинет пользователя с функциями просмотра данных энергопотребления согласно диаграмме прецедентов.

## Что было реализовано

### 1. ✅ Система аутентификации
- **LoginWindow.xaml/cs** - окно входа с авторизацией
- **RegisterWindow.xaml/cs** - окно регистрации новых пользователей
- **Проверка учетных данных** через БД

### 2. ✅ Модель данных User
- **Models/User.cs** - модель пользователя с полями:
  - UserID
  - Username (уникальное имя)
  - Password
  - Email
  - FullName
  - IsAdmin (флаг администратора)

### 3. ✅ База данных
- Создана таблица **[User]** в БД EnergyManagement
- Добавлены 3 тестовых пользователя:
  - `admin` - администратор
  - `user1` - обычный пользователь
  - `user2` - обычный пользователь

### 4. ✅ DatabaseService расширен
Добавлены методы:
- `AuthenticateUser()` - проверка логина/пароля
- `GetUserByID()` - получение пользователя по ID
- `CreateUser()` - создание нового пользователя
- `GetRoomsByBuilding()` - получение помещений здания
- `GetAllEnergyConsumption()` - alias для совместимости
- `GetEnergyReportByBuilding()`, `GetEnergyReportByRoom()`, `GetEnergyReportByTimePeriod()` - методы отчетов

### 5. ✅ Личный кабинет пользователя (UserWindow)
**Главное окно с 5 вкладками:**

#### 📦 Вкладка 1: "Здания"
- Отображение полного списка всех зданий
- Читаемые данные: ID, название, адрес
- Только просмотр (без редактирования)

#### 🏢 Вкладка 2: "Помещения"
- Список помещений с возможностью фильтрации
- **Фильтр по зданиям** - выборочное отображение
- **Кнопка "Показать все"** - отображение всех помещений
- Информация: ID, этаж, категория, здание

#### ⚡ Вкладка 3: "Энергопотребление"
- Просмотр данных энергопотребления
- **Фильтры:**
  - По зданию
  - По помещению (автообновляется)
  - По периоду времени
- Информация: помещение, дата, период, значение энергии

#### 🔍 Вкладка 4: "Поиск данных"
- Расширенный поиск с фильтрацией
- **Критерии:**
  - По зданию
  - От даты
  - До даты
- Результаты поиска в таблице

#### 📊 Вкладка 5: "Отчеты"
- Генерирование 3 типов отчетов:
  1. По зданиям
  2. По помещениям
  3. По периодам времени

### 6. ✅ Изменения в App.xaml
- `StartupUri` изменен с `MainWindow.xaml` на `LoginWindow.xaml`
- Теперь приложение запускается с экрана входа

### 7. ✅ Логика входа
- **Администратор** (IsAdmin=true) → открывается MainWindow (админ-панель)
- **Пользователь** (IsAdmin=false) → открывается UserWindow (личный кабинет)

### 8. ✅ Документация
- **USER_CABINET_GUIDE.md** - руководство пользователя
- Описание всех функций и возможностей
