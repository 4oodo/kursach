## 🚀 Тестирование приложения

### 1️⃣ Вход в систему как администратор

1. Запустите приложение
2. В окне входа введите:
   - **Логин:** `admin`
   - **Пароль:** `admin123`
3. Нажмите "Вход"
4. Должно открыться окно **MainWindow** (админ-панель) с вкладками:
   - Здания
   - Категории
   - Периоды времени
   - Помещения
   - Энергопотребление
   - SQL консоль
   - Отчеты
   - **Управление ключами администратора** (новая вкладка)

### 2️⃣ Вход в систему как обычный пользователь

1. В окне входа введите:
   - **Логин:** `user1`
   - **Пароль:** `user123`
2. Нажмите "Вход"
3. Должно открыться окно **UserWindow** (личный кабинет) с вкладками:
   - Здания (просмотр)
   - Помещения (просмотр с фильтром)
   - Энергопотребление (просмотр с фильтром)
   - Поиск
   - Отчеты

### 3️⃣ Регистрация нового пользователя

1. В окне входа нажмите "Регистрация"
2. Введите данные:
   - **Имя пользователя:** `newuser` (или другое)
   - **Пароль:** минимум 6 символов
   - **Подтверждение пароля:** повторите пароль
3. **БЕЗ** чекбокса администратора - просто нажмите "Зарегистрироваться"
4. Должно появиться сообщение об успешной регистрации
5. Закройте окно и войдите как новый пользователь

### 4️⃣ Регистрация администратора

1. В окне входа нажмите "Регистрация"
2. Введите данные:
   - **Имя пользователя:** `newadmin` (или другое)
   - **Пароль:** минимум 6 символов
   - **Подтверждение пароля:** повторите пароль
3. **Установите чекбокс** "Зарегистрироваться как администратор"
4. Поле "Ключ администратора" станет видимым
5. Введите один из валидных ключей:
   - `ADMIN_KEY_2024_001` ✅
   - `ADMIN_KEY_2024_002` ✅
6. Нажмите "Зарегистрироваться"
7. После успешной регистрации войдите с новыми учетными данными
8. Должна открыться админ-панель (MainWindow)

### 5️⃣ Управление ключами администратора

**Только для администраторов:**

1. Войдите как `admin` / `admin123`
2. Откройте вкладку "Управление ключами администратора"
3. Вы должны увидеть список существующих ключей в DataGrid
4. Функции:
   - **Добавить ключ:** введите значение (мин. 8 символов) и нажмите "Добавить"
   - **Деактивировать ключ:** выберите ключ и нажмите "Деактивировать"
   - **Удалить ключ:** выберите ключ и нажмите "Удалить"
   - **Обновить список:** нажмите "Обновить"

## 🔍 Диагностика проблем

### Если при регистрации выдается ошибка подключения:

**Проверьте:**

1. **SQL Server запущен:**
   ```powershell
   Get-Service -Name "MSSQLSERVER" | Select-Object Status
   ```
   Должно быть: `Status : Running`

2. **БД существует:**
   ```powershell
   sqlcmd -E -C -S localhost -Q "SELECT name FROM sys.databases WHERE name = 'EnergyManagement'"
   ```

3. **Таблицы созданы:**
   ```powershell
   sqlcmd -E -C -S localhost -d EnergyManagement -Q "SELECT COUNT(*) FROM information_schema.tables WHERE table_schema = 'dbo'"
   ```

4. **Пользователи в БД:**
   ```powershell
   sqlcmd -E -C -S localhost -d EnergyManagement -Q "SELECT * FROM dbo.[User]"
   ```

### Если не работает вход:

1. Проверьте, что пароль вводится правильно (учитывайте регистр)
2. Пароли хешируются SHA256, поэтому `admin123` → `8c6976e5b5410415bde908bd4dee15dfb167a9c873fc4bb8a81f6f2ab448a918`
3. Проверьте в Output окне Debug сообщения об ошибках

## 📝 Дополнительная информация

### Строка подключения:

```
Data Source=localhost;
Initial Catalog=EnergyManagement;
Integrated Security=True;
Connect Timeout=30;
Encrypt=False;
TrustServerCertificate=True
```

### Встроенная безопасность (Windows Auth):

Приложение использует интегрированную безопасность Windows (`Integrated Security=True`), это значит:
- Подключение к БД идет от имени текущего пользователя Windows
- Не требуется вводить пароль БД
- SQL Server должен быть настроен на прием Windows аутентификации

### Хеши паролей:

Пароли пользователей хешируются перед сохранением:
- **Алгоритм:** SHA256
- **Метод:** `PasswordHasher.HashPassword(password)`
- Пример: `admin123` → `8c6976e5b5410415bde908bd4dee15dfb167a9c873fc4bb8a81f6f2ab448a918`
