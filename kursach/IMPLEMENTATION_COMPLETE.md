## 📋 Что было реализовано

### 1. ✅ Расширенная система ролей и ключей

#### Таблицы БД:
- **Role** - таблица ролей (User, Admin)
- **AdminKey** - таблица для управления ключами администраторов
- **User** - обновленная таблица пользователей с полями RoleID

#### Модели данных:
- `Role.cs` - модель роли
- `AdminKey.cs` - модель ключа администратора
- `User.cs` - обновленная модель с RoleID

---

### 2. ✅ Система аутентификации

#### LoginWindow
- Окно входа с валидацией учетных данных
- Проверка пароля через хеширование SHA256
- Маршрутизация пользователей:
  - **Admin** → MainWindow (админ-панель)
  - **User** → UserWindow (личный кабинет)

#### RegisterWindow
- **Регистрация обычного пользователя**
  - Проверка уникальности имени
  - Валидация пароля (минимум 6 символов)
  - Подтверждение пароля

- **Регистрация администратора**
  - Чекбокс "Зарегистрироваться как администратор"
  - Поле ввода ключа администратора
  - **Валидация ключа** против таблицы AdminKey
  - Присвоение роли Admin только при вводе корректного ключа

#### Безопасность
- **PasswordHasher.cs** - утилита для хеширования паролей SHA256
- Пароли хранятся в виде хешей, не в открытом виде
- Ключи администратора проверяются в БД

---

### 3. ✅ Личный кабинет пользователя (UserWindow)

Пользователи имеют доступ к 5 вкладкам:

1. **Здания** - просмотр всех зданий
2. **Помещения** - просмотр помещений с фильтром по зданиям
3. **Энергопотребление** - просмотр данных с фильтрацией
4. **Поиск** - расширенный поиск по зданию, дате
5. **Отчеты** - генерирование аналитических отчетов

---

### 4. ✅ Административная панель (MainWindow)

Администраторы имеют доступ к управлению:

1. **Зданиями** - CRUD операции
2. **Категориями** - CRUD операции
3. **Периодами времени** - CRUD операции
4. **Помещениями** - CRUD операции
5. **Энергопотреблением** - CRUD операции
6. **SQL консолью** - выполнение произвольных запросов
7. **Отчетами** - формирование аналитических отчетов

#### 🔑 **НОВОЕ: Управление ключами администратора**
- Просмотр списка всех ключей
- **Статус ключа**: активен/неактивен
- **Добавление нового ключа**
  - Проверка уникальности
  - Минимум 8 символов
- **Деактивация ключа** - отключение без удаления
- **Удаление ключа** - полное удаление из БД
- Интерактивные кнопки с подтверждением

## 📝 SQL скрипты

В Database_Setup.sql добавлены:

```sql
-- Таблица ролей
CREATE TABLE dbo.Role (
	RoleID INT PRIMARY KEY IDENTITY(1,1),
	RoleName NVARCHAR(100) NOT NULL UNIQUE
);

-- Таблица ключей администратора
CREATE TABLE dbo.AdminKey (
	KeyID INT PRIMARY KEY IDENTITY(1,1),
	AdminKeyValue NVARCHAR(100) NOT NULL UNIQUE,
	IsActive TINYINT NOT NULL DEFAULT 1
);

-- Обновленная таблица пользователей
CREATE TABLE dbo.[User] (
	UserID INT PRIMARY KEY IDENTITY(1,1),
	Username NVARCHAR(45) NOT NULL UNIQUE,
	PasswordHash NVARCHAR(4000) NOT NULL,
	RoleID INT NOT NULL,
	FOREIGN KEY (RoleID) REFERENCES dbo.Role(RoleID)
);
```
